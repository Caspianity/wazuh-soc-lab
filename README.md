# wazuh-soc-lab
This project demonstrates the deployment of a Home Security Operations Center (SOC) lab using Wazuh, an open-source SIEM platform. The primary goal of this lab is to simulate a real-world security monitoring environment where system activity and logs can be collected, analyzed, and visualized through a centralized dashboard. By building this setup, I was able to gain hands-on experience in log analysis, threat detection, and the overall workflow of a SOC analyst.

The lab environment consists of a Wazuh server deployed using Docker within Windows Subsystem for Linux (WSL), alongside a Kali Linux virtual machine configured as a monitored endpoint. The Wazuh server is composed of three main components: the manager, which processes and analyzes incoming data from agents; the indexer, which stores and indexes security events for efficient searching; and the dashboard, which provides a web-based interface for monitoring alerts and system status. The Kali Linux machine runs the Wazuh agent, which continuously collects system logs and sends them to the Wazuh manager for analysis. This setup allows for real-time visibility into endpoint activity and simulates how security teams monitor infrastructure in production environments.

To deploy the environment, Docker was installed and configured on WSL, including the adjustment of the required Linux kernel parameter vm.max_map_count to ensure compatibility with the Wazuh indexer. The official Wazuh Docker repository was cloned, SSL certificates were generated for secure communication between components, and the full Wazuh stack was deployed using Docker Compose. The containers were verified to be running properly, and the Wazuh dashboard was accessed through a web browser. On the endpoint side, the Wazuh agent was installed on Kali Linux, configured with the manager’s IP address, and successfully enrolled into the system, allowing it to send telemetry data such as logs, file integrity events, and system activity.
